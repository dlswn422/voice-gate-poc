# 카드/결제 실패 (정산기/키오스크/출구정산) - PAYMENT_ISSUE

## 대표 키워드
- 결제 안돼요
- 카드 결제 실패
- 승인 실패
- 단말 오류
- 통신 오류
- 정산기
- 키오스크
- 출구 정산기
- IC칩
- 카드 인식 불가
- 버튼 무반응
## 필수 슬롯 (required_slots)
- symptom: 어떤 현상인지(승인 실패/카드 인식 불가/버튼 무반응/정산 반영 문제 등)
- where: 어디에서 결제했는지(출구 정산기/사전 정산기(키오스크)/모바일 등)
## 추가 슬롯 (optional_slots)
- error_message: 화면 오류 문구/코드 (있으면 매우 중요)
- attempted: 이미 해본 조치(재시도/다른 카드/다른 결제수단)
- card_or_device: 카드 문제인지/단말(기기) 문제인지 단서
## 첫 질문 (ASK_1)
결제에서 어떤 현상이 발생하시나요? (승인 실패 / 카드 인식 불가 / 결제 버튼 무반응 / 정산 반영 문제 중 어떤 건가요?)
## 슬롯 추출 가이드
- 사용자가 '승인이 안 돼요/승인 실패' → symptom=승인 실패
- '카드가 안 먹어요/인식이 안 돼요' → symptom=카드 인식 불가
- '버튼이 눌러도 반응이 없어요/화면이 멈췄어요' → symptom=버튼 무반응 또는 단말 먹통
- '결제했는데 반영이 안 돼요/요금이 그대로예요' → symptom=정산 반영 문제(할인/무료시간 포함 여부도 확인)
- where는 사용자의 표현에서 '출구/나가는 곳'이면 출구 정산기, '키오스크/정산기'면 사전 정산기, '앱/QR'면 모바일
- error_message는 화면에 보이는 문구를 그대로 받기(예: '통신 오류', '서버 연결 실패', '승인 실패')
## 진단 분기 (결정 규칙)
1) error_message에 '통신/연결/서버/오프라인'이 포함되면 → 네트워크/단말 장애 가능성 ↑ (network_terminal_down 참고)
2) symptom=카드 인식 불가면 → 카드 삽입/방향/IC칩 상태 및 다른 카드로 분리 진단
3) symptom=버튼 무반응/화면 멈춤이면 → 단말(기기) 이상 가능성 ↑ (kiosk_ui_device_issue 참고)
4) symptom=정산 반영 문제면 → 할인/무료시간 미적용 가능성도 함께 확인 (discount_free_time_issue 참고)
## 조치 순서 (우선순위)
1) 결제를 어디에서 했는지 확인 후(출구 정산기/키오스크/모바일) 동일 위치에서 한 번 더 재시도(10~20초 후)
2) 카드 결제라면 카드 방향/삽입 상태(IC칩/마그네틱) 확인 후 재시도
3) IC칩(금속면) 오염이면 닦은 뒤 재시도
4) 가능하면 다른 카드 또는 다른 결제수단으로 시도
5) 화면에 통신/서버 오류가 반복되면 다른 단말/차로 이용을 안내하고 관리자 호출
## 해결 안내 문장 템플릿 (SOLVE_TEMPLATE)
- (승인 실패) 승인 실패는 카드사/카드 상태 또는 단말 통신 상태 문제일 수 있어요. 카드 방향/IC칩을 확인하고 다른 카드로도 재시도해 주세요.
- (통신 오류) 화면에 통신/서버 오류가 반복되면 단말 또는 네트워크 장애 가능성이 있어요. 다른 정산기/차로로 시도하고, 계속되면 관리자를 호출해 주세요.
- (인식 불가) 카드 인식이 안 되면 IC칩 방향/오염 여부를 확인하고, 다른 카드로도 시도해 주세요.
- 마지막: 다른 문제가 더 있으신가요?
## 에스컬레이션 조건
- 동일 단말에서 2~3회 재시도 후에도 동일 오류 반복
- '통신/연결/서버/오프라인' 오류가 지속
- 화면 멈춤/버튼 무반응 등 단말 자체 이상이 의심될 때
